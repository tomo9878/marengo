## 🔴 重要（ゲームバランスに直結）

- [x] **1. 突撃敗北後：同アプローチ経由の攻撃・行軍禁止**
  - ルール：攻撃側が敗北した場合、同一ターン中、同じアプローチを通過しての攻撃または行軍ができない
  - 対象ファイル：`MoveValidator.js`、`GameState.js`（状態記録）

- [x] **2. 突撃勝利後：防御側ロケールへの道路行軍禁止**
  - ルール：突撃が勝利した場合、以降同一ターン中、どのアプローチからも防御側ロケールへ道路行軍で進入できない
  - 対象ファイル：`MoveValidator.js`、`GameState.js`（状態記録）

- [x] **3. 急襲士気2トークン条件の完成**
  - ルール：防御側アプローチが広い AND 攻撃側駒が2個以上 AND このターンこのアプローチを越えた最初の急襲 → 2トークン（現在「最初の急襲」チェックが未実装）
  - 対象ファイル：`CombatResolver.js`、`GameState.js`（ターン内急襲履歴）

- [x] **4. 急襲の騎兵障害物チェック（攻撃側）**
  - ルール：防御アプローチに騎兵障害物がある場合、攻撃側に歩兵が含まれていなければならない
  - 対象ファイル：`MoveValidator.js`（getLegalRaids）

- [x] **5. フランス軍の混乱（6:00AMラウンド）**
  - ルール：ラウンド1のみ、フランス軍はいかなる場合もアプローチをブロックできない
  - 対象ファイル：`MoveValidator.js`（防御行軍・悪路行軍のブロック判定）

## 🟡 士気・クリーンアップ系

- [x] **6. 士気クリーンアップ「最後に敵がいた」条件**
  - ルール：現在敵が占拠していなくても、最後にそのロケールに存在した駒が敵だった場合もトークンを除去
  - 対象ファイル：`MoraleManager.js`（moraleCleanup）、`GameState.js`（占拠履歴の記録）

- [x] **7. FRANCE_MORALE_RECOVERY「直前の相手ターンに移動されたトークンは選択不可」**
  - ルール：フランス回収時、直前のオーストリアターン中に別ロケールへ移動されたトークンは選択できない
  - 対象ファイル：`MoraleManager.js`（getRecoverableTokens）、`GameState.js`（相手ターン移動履歴）

## 🟡 道路行軍・横断系

- [x] **8. 横断の複数対応**
  - ルール：隣接ロケール間に2本の道路がある場合、各道路がそれぞれ独立した横断を持ち交通制限も独立する
  - 対象ファイル：`MapGraph.js`、`MoveValidator.js`、`data/map.json`（crossingsデータ整備）
  - 注記：既存のコードとデータ（locales 9-10）で正しく実装済みを確認

- [x] **9. 道路行軍による急襲：通過する横断の指定**
  - ルール：道路行軍による急襲の宣言時、追加で通過する横断を指定する必要がある。1つの横断を道路行軍による急襲に用いることができる回数はターン中1回まで
  - 対象ファイル：`MoveValidator.js`（getLegalRoadMoves で raidTargetLocaleId/raidCrossingId 生成）、`TurnManager.js`（executeMarchRaid）、`GameState.js`（roadMarchRaidCrossings）
  - 注記：エンジン完全実装済み。クライアント側は通常の道路行軍アクションとして選択される（UI上の明示的な横断選択ステップは未実装）

- [x] **10. 増援進入時の交通制限**
  - ルール：マップの端はその道路の最初の横断とみなされ交通制限が適用される
  - 対象ファイル：`MoveValidator.js`（getLegalEntryActions）
